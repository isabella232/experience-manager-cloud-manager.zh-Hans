# 自定义权限 {#custom-permissions}

了解如何使用自定义权限创建新的自定义权限配置文件，以及可配置的权限来限制对Cloud Managers用户的程序、管道和环境的访问。

>[!NOTE]
>
>此功能仅适用于[早期采用者计划。](/help/release-notes/current.md#early-adoption)

## 简介 {#introduction}

Cloud Manager具有一组预定义角色，这些角色控制对Cloud Manager各种功能的访问：

* 业务负责人
* 项目管理员
* 部署管理员
* 开发人员

自定义权限允许用户创建新的自定义权限配置文件，可配置权限以限制Cloud Manager用户对程序、管道和环境的访问。

>[!TIP]
>
>有关预定义角色的详细信息，请参阅文档 [基于角色的权限。](/help/requirements/role-based-permissions.md)

## 使用自定义权限 {#using}

要创建并使用您自己的自定义权限，需要执行三个步骤：

1. [创建新的产品配置文件。](#create)
1. [将自定义权限分配给新的产品配置文件。](#assign-permissions)
1. [将用户分配到新的产品配置文件。](#assign-users)

本节将详细介绍这些步骤。 您可能会发现参阅 [术语](#terms) 和 [可配置权限](#configurable-permissions) 创建自己的自定义权限时显示的区域。

>[!NOTE]
>
>您必须具有Admin Console中的产品管理员权限才能创建新配置文件并管理Cloud Manager的权限。

### 创建新的产品配置文件 {#create}

您首先必须创建新的产品配置文件，然后才能为其分配自定义权限。

1. 登录Cloud Manager，网址为 [my.cloudmanager.adobe.com](https://my.cloudmanager.adobe.com/)

1. 选择产品 **AEM Managed Services**.

1. 搜索名称与模式匹配的和实例 `*-cloud-manager` 和点按或单击以管理用户和权限。

1. 您将被重定向到 **产品** Admin Console选项卡中，您可以在此处管理cloud manager的用户和权限。 在Admin Console中，点按或单击 **新建配置文件** 按钮。

![“新建配置文件”按钮](/help/assets/admin-console-new-profile.png)

1. 提供有关用户档案的一般详细信息。

   * **产品配置文件名称**  — 配置文件的描述性名称
   * **显示名称**  — 将显示在UI（选项）中的缩写名称
   * **描述**  — 配置文件的信息性描述，用于解释其用途（可选）
   * **通过电子邮件通知用户**  — 选中后，用户将在添加到此配置文件或从中移除时收到电子邮件通知。

1. 点击或单击 **保存** 完成后。

新产品配置文件已保存，并显示在Admin Console的产品配置文件列表中。

### 将自定义权限分配给配置文件 {#assign-permissions}

现在您有了新的产品配置文件，您可以为其分配自定义权限。

1. 在Admin Console中，点按或单击 [您刚刚创建的新产品配置文件。](#create)

1. 在打开的窗口中，选择 **权限** 选项卡以查看可编辑权限列表。

   ![可编辑的权限](/help/assets/permissions-tab.png)

1. 点按或单击 **编辑** 用于编辑该文档的权限的链接。

1. 此 **编辑权限** 窗口打开。
   * 您在上一步中选择的权限将在左列中进行选择。
   * 可用于分配权限的权限项位于中间列，其标签为 **可用权限** 项目。
   * 分配的权限项位于右侧列中，标记为 **包含的权限项**.

   ![编辑权限项](/help/assets/edit-permission-items.png)

1. 点按或单击加号(`+`)图标以将其添加到列 **包含的权限项**.

   * 点按或单击 `i` 图标来了解权限项的更多信息。

1. 点按或单击 **全部添加** 顶部按钮 **可用权限** 列以添加所有权限。 同样，点按或单击 **全部移除** 以删除所有以前选定的权限。

1. 点击或单击 **保存** 当您完成新产品配置文件的权限项定义时。

现在将保存您的新产品用户档案及其自定义权限。

### 将用户分配给自定义权限 {#assign-users}

您现在可以将用户分配给您使用自定义权限创建的新产品用户档案。

1. 在Admin Console中，点按或单击 [您刚刚为其分配了自定义权限的新产品配置文件。](#assign-permissions)

1. 在打开的窗口中，选择 **用户** 选项卡。

1. 点按或单击 **添加用户** 按钮并将用户分配给您具有自定义权限的新产品配置文件。

请参阅部分 **将用户和用户组添加到产品配置文件** 文档的 [管理企业用户的产品配置文件](https://helpx.adobe.com/enterprise/using/manage-product-profiles.html) 以了解有关如何使用Admin Console的更多详细信息。

## 可配置权限 {#configurable-permissions}

以下权限可用于创建自定义配置文件。

| 权限 | 描述 |
|---|---|
| 项目访问 | 允许用户访问程序 |
| 项目编辑 | 允许用户编辑项目 |
| 管道创建 | 允许用户创建新管道 |
| 管道删除 | 允许用户删除管道 |
| 管道编辑 | 允许用户编辑管道 |
| 生产部署批准/拒绝 | 允许用户批准或拒绝生产部署步骤 |
| 管道执行取消 | 允许用户取消管道执行 |
| 管道执行开始 | 允许用户启动新的管道执行 |
| 覆盖/拒绝重要量度失败 | 允许用户覆盖/拒绝重要量度失败 |
| 生产部署计划 | 允许用户计划生产部署步骤 |
| 存储库信息访问 | 允许用户访问存储库信息并生成访问密码 |

### 组织级别的权限 {#organization-level}

组织级别的权限是指始终在组织中的所有项目之间授予的权限。

以下权限是组织级别的权限：

* **存储库信息访问** 此租户/组织级别权限允许用户生成用户名、密码和存储库URL，用于访问和参与客户项目。
   * 存储库访问的用户名和密码在组织中的所有存储库中都是通用的，但每个项目的存储库URL都是唯一的。
   * 请参阅文档 [源代码存储库](/help/requirements/source-code-repository.md) 以了解更多信息。

## 术语 {#terms}

以下术语用于创建和管理自定义权限和预定义角色。

| 术语 | 描述 |
|---|---|
| 预定义权限 | 预定义的角色，如 **业务负责人**， **部署管理员**，等等。 以管理Cloud Manager的各种功能。 有关预定义角色的详细信息，请参阅文档 [基于角色的权限。](/help/requirements/role-based-permissions.md) |
| 自定义权限 | Cloud Manager功能，允许用户创建权限配置文件以定义用于管理Cloud Manager所支持功能的角色 |
| 权限配置文件 | 在Admin Console中创建，用于管理可配置的权限，这些权限将适用于属于权限配置文件的用户 |
| 可配置权限 | 可在权限配置文件中配置的Cloud Manager权限 |
| 权限项 | 可对其应用权限的项目、环境或管道资源 |

权限项是指将应用权限的范围。 通常，它将是以下内容之一。

| 权限项类型 | 示例 | 描述 |
|---|---|---|
| 组织 | organization：companyA | 组织的所有适用资源。 资源可以是项目、环境或管道。 如果用户为任何权限添加组织，则该组织中的所有新资源也将具有该权限。 |
| 项目 | 项目A | 项目的所有适用资源 |
| 环境 | 项目A ：环境 | 适用于特定环境 |
| 管道 | 项目A：管道 | 适用于特定管道 |

## 限制 {#limitations}

使用自定义权限时，请牢记以下限制。

* A [可用的权限集有限](#configurable-permissions) 用于创建自定义用户档案。
* 项目、环境、管道等资源。 在Cloud Manager中创建的权限可能需要两分钟才能显示在Admin Console中以进行权限配置。
* 在自定义权限服务无法响应的极少数情况下，预定义用户档案仍然可用，并且预定义用户档案中的用户仍然具有适当的访问权限。

## 常见问题解答 {#faq}

### 哪些权限配置文件是预定义的权限配置文件？

* 业务负责人
* 项目管理员
* 部署管理员
* 开发人员

有关预定义角色的详细信息，请参阅文档 [基于角色的权限。](/help/requirements/role-based-permissions.md)

### 随着自定义用户档案的推出，预定义的权限用户档案会发生什么情况？

默认产品配置文件和Cloud Manager角色的工作方式与之前相同。

### 我是否可以编辑预定义的权限配置文件？

否，默认配置文件不可编辑。 您无法将权限添加到默认权限配置文件或从中移除权限。 您只能在预定义的用户档案中添加或删除用户。

### 既然自定义配置文件现已可用，我应该删除预定义的权限配置文件吗？

不得从Admin Console中删除预定义的权限配置文件。

### 我可以将用户添加到多个权限配置文件吗？

可以，用户可以包含在多个配置文件中，包括预定义和自定义权限配置文件。 当用户被分配给多个配置文件时，所有分配的权限配置文件的组合权限将对该用户可用。

### 如果用户有权编辑环境/管道，但无权访问包含环境/管道的项目，会发生什么情况？

在这种情况下，如果用户没有，他们将无法访问环境或管道。 **项目访问** 包含环境或管道的权限。

### 如果我在同一IMS组织中同时具有AEMas a Cloud Service和AMS程序，会发生什么？ 我是否可以通过一个配置文件管理权限？ {#ams-and-aemaacs}

您应该为每个产品类型创建单独的配置文件(即，一个用于AEM作为Cloud Service，一个用于AdobeManaged Services或AMS)。
